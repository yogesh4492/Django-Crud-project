Node.js uses the V8 JavaScript engine from Chrome to execute JavaScript outside the browser.

Event-Driven and Non-Blocking I/O

Node.js uses event-driven architecture and non-blocking input/output, which makes it suitable for high-performance applications.

Instead of waiting for operations to complete, Node.js continues executing other code and uses callbacks to handle results when they're ready.

// Non-blocking file read
fs.readFile('file.txt', function(err, data) {
console.log(data);
});
console.log('This runs immediately');

Single-Threaded Event Loop

Node.js uses a single-threaded event loop, which can handle thousands of concurrent connections efficiently without creating multiple threads.

const http = require('http');

const server = http.createServer(function(req, res) {
res.writeHead(200);
res.end('Hello Node.js');
});

server.listen(3000);

Basic Node.js Example

const express = require('express');
const app = express();

app.get('/', function(req, res) {
res.send('Hello World!');
});

app.listen(3000, function() {
console.log('Server running on port 3000');
});
